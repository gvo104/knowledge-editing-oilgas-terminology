# Текущее состояние по цели исследования

Сейчас мы на этапе подготовки рабочего контура экспериментов. Базовая инфраструктура уже собрана: окружение работает, `Qwen2.5-3B` загружена локально, тестовые прогоны `FT`, `LoRA`, `ROME`, `MEMIT` на простом факте проходят, а формат данных для редактирования унифицирован под все методы.

Относительно целей из `my_exp/reserch.txt` это значит следующее:
- этап "проверить, что методы вообще запускаются и редактирование работает" уже пройден
- этап "подготовить универсальный формат фактов и вопросов" тоже в основном закрыт
- этап "собрать полноценный биомедицинский набор для честного сравнения" еще не завершен
- этап "knowledge probing до/после на реальных данных" пока не начат в полном виде

Что уже сделано:
- собран `jsonl`-формат с полями для `rewrite`, `rephrase`, `locality`, `portability`
- доработан `my_exp/data_processing`, чтобы из локального PubTator можно было собрать edit-dataset
- подготовлен CLI для экспорта набора кейсов из PubTator

Ближайшие шаги:
- выбрать, на каких именно биомедицинских фактах сравнивать методы
- собрать локальный набор кейсов не только для smoke-test, а для серии экспериментов
- подключить запуск методов не на одном кейсе, а на `jsonl`-наборе
- добавить автоматическую агрегацию метрик до/после редактирования

Предложения:
- не ограничиваться отношением `entity -> PMID`, а по возможности перейти к более содержательным биомедицинским фактам
- сначала сделать маленький, но чистый benchmark на 50-200 кейсов
- сравнивать методы минимум по четырем вещам: успешность правки, устойчивость на парафразах, сохранение соседних фактов, скорость/стоимость прогона
- отдельно держать два сценария: одиночное редактирование и последовательное редактирование
